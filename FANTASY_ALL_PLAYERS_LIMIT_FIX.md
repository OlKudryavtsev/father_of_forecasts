# Fantasy all players limit fix

## Проблема

В Fantasy в списке сборных могли отсутствовать некоторые команды, например `Шотландия`, хотя игроки есть в таблице `fantasy_players`.

Причина: Mini App загружал игроков так:

```http
GET /api/webapp/fantasy/players?limit=1000
```

А endpoint ограничивал максимум:

```python
limit <= 1000
```

После повторных импортов и расширенных составов общее число активных игроков может быть больше 1000.
Из-за сортировки часть сборных оказывалась за пределами первых 1000 игроков.
Frontend строит список доступных сборных для конкретной позиции по уже загруженному списку игроков, поэтому такие сборные исчезали из выбора.

## Что исправлено

1. Backend endpoint `/api/webapp/fantasy/players` теперь допускает:

```python
limit <= 5000
```

и по умолчанию возвращает до 5000 игроков.

2. Frontend теперь запрашивает:

```http
/api/webapp/fantasy/players?limit=5000
```

Этого достаточно для всех 48 сборных даже при расширенных составах.

Миграции не нужны.
