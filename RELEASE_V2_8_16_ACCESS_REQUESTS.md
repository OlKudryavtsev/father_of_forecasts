# v2.8.16 — Заявки и подтверждение пользователей

## Что изменилось

- Новые пользователи после `/start` переводятся в статус `pending`.
- Администраторам отправляется заявка с inline-кнопками:
  - `✅ Одобрить`
  - `❌ Отклонить`
- После одобрения пользователь:
  - получает статус `approved`;
  - добавляется в лигу `Отец прогнозов`;
  - если пришел по invite-code, дополнительно добавляется в приглашенную лигу.
- Pending/rejected/blocked пользователи не могут пользоваться прогнозами, рейтингами, матч-центром и PWA до одобрения.
- `/start app` и базовые команды не ломаются, но веб-приложение для pending-пользователей возвращает отказ доступа.

## Миграция

```bash
psql "$DATABASE_URL" -f db/migrations/012_access_requests_approval.sql
```

## Совместимость

- Текущие участники не теряются и остаются `approved`.
- Текущие прогнозы, очки, рейтинги и fantasy-данные не пересоздаются.
- UX для уже одобренных пользователей не меняется.
