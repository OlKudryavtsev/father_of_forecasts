# Отец прогнозов — v3.4.0

## Что исправлено

1. **Дубли Telegram-сообщений**
   - доставка теперь сначала атомарно резервирует адресата в БД, а только затем отправляет сообщение;
   - это защищает от параллельной работы двух процессов/реплик Railway;
   - уникальность обеспечивается на уровне PostgreSQL.

2. **Пропуски и нарушенный порядок событий**
   - каждое событие вопроса содержит снимок текста, номера раунда, номера вопроса, таймера и стадии «Обратного отсчёта»;
   - Telegram больше не зависит от текущего статуса вопроса на момент отправки;
   - «Обратный отсчёт» не может прислать третью подсказку вместо первой.

3. **Полная коммуникация игры**
   - старт квиза;
   - старт каждого раунда;
   - каждый вопрос с номером раунда и номером вопроса;
   - раскрытие правильного ответа;
   - окончание раунда с промежуточной таблицей;
   - окончание квиза с победителем и итоговой таблицей.

4. **Таймеры по формату**
   - 4 варианта: 30 сек.;
   - бинарные: 20 сек.;
   - «Своя игра»: 50 сек.;
   - «Один из двух»: 45 сек.;
   - «Что? Где? Когда?»: 75 сек.;
   - «Обратный отсчёт»: 25 / 25 / 35 сек.;
   - «Сто к одному»: 50 сек.

   В форме создания квиза таймеры можно изменить до старта. Существующие квизы, созданные ранее, используют эти значения по умолчанию.

5. **Порядок раундов при автосборке из банка**
   1. Кто хочет стать миллионером?
   2. Правда или ложь
   3. Больше или меньше
   4. Да или нет
   5. Обратный отсчёт
   6. Своя игра
   7. Один из двух
   8. Что? Где? Когда?
   9. Сто к одному

## Обязательная миграция

Перед деплоем выполните:

```sql
\i db/migrations/028_quiz_delivery_flow_and_timers.sql
```

## Проверка

1. Создайте новый квиз из девяти тестовых вопросов.
2. В Telegram должны прийти только по одному сообщению каждого вида.
3. В чате лиги проверьте порядок: старт квиза → старт раунда → вопрос → ответ → завершение раунда → следующий раунд.
4. Для «Обратного отсчёта» убедитесь, что подсказки идут 1 → 2 → 3 подряд, без сообщений другого раунда между ними.
5. В PWA проверьте различный таймер для разных форматов.
