# v2.8.31 — Детали матча и единый backend-кэш

## Что добавлено

- В Матч-центре матч открывается по нажатию на команды/счет или кнопке «Детали матча».
- Новый экран матча содержит вкладки: **Обзор**, **События**, **Бомбардиры**, **Статистика**, **Составы**.
- В обзоре показываются счет, стадион, город, арбитр, турнирный раунд, голы и CTA прогноза до стартового свистка.
- В событиях: голы, карточки, замены и VAR с минутой и ассистом при наличии.
- В бомбардирах: авторы голов, минуты, количество голов и фото игрока при наличии у API-Football.
- В статистике: владение, удары, удары в створ, угловые, фолы, карточки, xG и другие показатели, когда их возвращает провайдер.
- В составах: схема, тренер, стартовый состав и раскрываемые запасные.

## Кэш API-Football

- Добавлена таблица `match_details_cache`: одна строка на матч с payload-ами fixture/events/statistics/lineups/players.
- PWA читает данные только через backend; прямых запросов из браузера к API-Football нет.
- Кэш обновляется по TTL: live — раз в минуту, перед матчем — раз в 15 минут, завершенный матч — раз в 12 часов.
- Добавлен background-loop прогрева для матчей за 2 часа до старта и до 6 часов после старта.
- Если API-Football временно недоступен, UI продолжает показывать уже сохраненные сведения и локальные данные матча.

## Деплой

После деплоя обязательно применить миграцию:

```bash
psql "$DATABASE_URL" -f db/migrations/016_add_match_details_cache.sql
```

Опциональные настройки:

```bash
MATCH_DETAILS_AUTOSYNC_ENABLED=true
MATCH_DETAILS_AUTOSYNC_INTERVAL_SECONDS=90
MATCH_DETAILS_LIVE_TTL_SECONDS=60
MATCH_DETAILS_UPCOMING_TTL_SECONDS=900
MATCH_DETAILS_FINISHED_TTL_SECONDS=43200
```
