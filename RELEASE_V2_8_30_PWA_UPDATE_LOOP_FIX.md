# v2.8.30 — PWA update loop fix

## Исправлено

- Исправлен зацикленный баннер «Доступна новая версия v2.8.28».
- Корневой файл `VERSION` синхронизирован с версией frontend/service worker: `2.8.30`.
- Баннер показывается только когда версия на сервере **строго новее**, а не при любом отличии версий.
- При старой или рассинхронизированной версии backend баннер автоматически скрывается.
- Обновление больше не снимает регистрацию Service Worker, чтобы не нарушать Web Push в установленном iOS PWA.
- Service Worker регистрируется с `updateViaCache: 'none'`.
- `sw.js`, `app-version.json` и manifest теперь отдаются с `no-store`/`no-cache`.
- После кнопки «Обновить» приложение загружает свежий `/app` с cache-busting параметром и выполняет навигацию через `location.assign`.
