# Квиз-баттл в групповом чате

`/quiz_battle` — режим серии вопросов для общего Telegram-чата.

## Пользовательский сценарий

1. В групповом чате вызвать команду:

```text
/quiz_battle
```

2. Бот предложит выбрать количество вопросов:

```text
3 вопроса | 5 вопросов | 10 вопросов
```

3. После выбора бот создает игру и автоматически отправляет первый вопрос.

4. На каждый вопрос дается 60 секунд.

5. Когда время вышло, бот показывает правильный ответ только если был хотя бы один правильный ответ. Если никто не отвечал или правильных ответов не было, правильный ответ не раскрывается.

6. Следующий вопрос появляется автоматически, даже если ответили не все.

7. После последнего вопроса бот показывает итоговую таблицу.

## Команды и callback-префиксы

Команда:

```text
/quiz_battle
```

Callback-префиксы:

```text
quiz_battle_size:
quiz_battle_answer:
```

Они разрешены в групповом чате через `GROUP_ALLOWED_COMMANDS` и
`GROUP_ALLOWED_CALLBACK_PREFIXES`.

## Отличие от `/quiz`

`/quiz` в группе запускает одиночный вопрос, который надо завершать командой:

```text
/quiz_finish
```

`/quiz_battle` запускает серию вопросов с автоматическим таймером. Завершать
каждый вопрос вручную не нужно.

## Таблицы БД

Для режима используются отдельные таблицы:

```text
group_quiz_games
group_quiz_game_questions
group_quiz_game_answers
```

SQL-миграция:

```text
db/migrations/001_add_quiz_battle.sql
```

## Ограничения текущей реализации

Таймер работает через `asyncio.create_task(...)` внутри процесса бота.

Это означает:

- если Railway перезапустит контейнер во время игры, активный баттл может оборваться;
- для дружеского чата это приемлемо;
- если нужна максимальная надежность, следующий этап — отдельный background-loop,
  который сам ищет открытые вопросы старше 60 секунд и закрывает их.

## Проверка после деплоя

В групповом чате:

```text
/quiz_battle
```

Выбрать `3 вопроса`, ответить на вопрос и дождаться автоматического перехода.

После трех вопросов должен появиться финальный результат.
