## Как применить к Railway PostgreSQL

1. Открой Railway → PostgreSQL → Query.
2. Выполни содержимое файла:

```text
db/migrations/001_add_quiz_battle.sql
```

3. После применения можно проверить схему:

```bash
python scripts/check_db_schema.py
```

## Как проверить в Telegram

В групповом чате:

```text
/quiz_battle
```

Дальше выбрать:

```text
3 вопроса
```

Ожидаемое поведение:

1. Бот отправляет вопрос 1/3.
2. Участники отвечают кнопками A/B/C/D.
3. После каждого принятого ответа бот пишет в чат: `✅ <Имя участника> сделал свой выбор`, не раскрывая вариант ответа.
4. Через 60 секунд бот показывает правильный ответ только при наличии хотя бы одного правильного ответа. Если никто не отвечал или правильных ответов не было, правильный ответ не раскрывается.
5. Автоматически появляется следующий вопрос.
6. После третьего вопроса бот показывает итоговую таблицу.

## BotFather

Добавить команду:

```text
quiz_battle - групповой квиз с таймером
```

## Важно

Таймер реализован через `asyncio.create_task(...)` в процессе бота. Если Railway
перезапустит контейнер во время активного квиза, серия может оборваться. Для
дружеского чата это приемлемо. Если нужна максимальная надежность, следующий этап —
отдельный background-loop, который закрывает просроченные вопросы по данным БД.

## Изменение в этой версии

После сохранения ответа в `/quiz_battle` бот отправляет публичное сообщение в групповой чат:

```text
✅ <Имя участника> сделал свой выбор
```

Это помогает сразу видеть, кто участвует в текущем вопросе, но не раскрывает выбранный вариант.
