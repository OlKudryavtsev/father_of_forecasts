# v3.3.1 — исправление загрузки тестовых вопросов квиза

## Причина

В v3.2.0 приложение получило типы `true_false`, `more_less` и `yes_no`, но в PostgreSQL остался исходный CHECK `chk_league_quiz_question_type` из миграции 023. Поэтому первый вопрос формата «Правда или ложь» не сохранялся.

## Что исправлено

Миграция `db/migrations/027_fix_league_quiz_question_types.sql` пересоздаёт CHECK и разрешает все актуальные типы:

- `choice_2`;
- `choice_4`;
- `true_false`;
- `more_less`;
- `yes_no`;
- `jeopardy`;
- `one_of_two`;
- `what_where_when`;
- `countdown`;
- `hundred_to_one`.

Загрузка банка выполняется в одной транзакции. Если операция не проходит, сервер откатывает её; частично созданные вопросы не сохраняются.

## Установка

1. Примените миграцию `027` в Railway PostgreSQL после миграций 023–026.
2. Задеплойте v3.3.1.
3. Обновите PWA.
4. Нажмите «Квиз → Управление квизом → Загрузить вопросы ЧМ-2026».

Ожидаемый результат: добавляется 9 одобренных вопросов, по одному на каждый поддержанный формат. Повторное нажатие не создаёт дублей.
