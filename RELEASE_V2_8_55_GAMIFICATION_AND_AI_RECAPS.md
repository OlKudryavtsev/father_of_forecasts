# v2.8.55 — Геймификация 1.0 и AI-итоги матчей

## Что добавлено

### Профиль и достижения

- У каждого участника появился динамический титул: например, `Хирург счёта`, `Чует исход`, `Железная дисциплина`, `На кураже`, `Тихий убийца`, `Лидер лиги`.
- Рядом с титулом показывается текущая форма: серия с очками, ожидание прогноза, лидерство или другой актуальный статус.
- Профиль теперь учитывает активную лигу из шапки приложения.
- Достижения стали многоуровневыми: Бронза, Серебро, Золото и Легенда.
- Добавлены достижения за точные счета, угаданные исходы, серии, дисциплину, идеальные игровые дни, уникальные попадания, лидерство и отсутствие пропусков.
- Прогресс достижений начинается с момента применения миграции `020_add_gamification_and_ai_messages.sql`. Старые результаты остаются в обычной статистике и влияют на титул, но не закрывают новую коллекцию сразу после установки.

### Личные сообщения после матча

- После завершения матча каждый участник получает отдельный итог по каждой своей лиге.
- В сообщении всегда есть проверяемые факты: результат, прогноз, начисленные очки, место/изменение места, серия и новые достижения.
- OpenAI генерирует только короткий комментарий в стиле Отца; модель не рассчитывает очки, позиции или достижения.
- Если OpenAI недоступен, отправляется встроенный шаблонный комментарий, поэтому уведомления не пропадут.

### Утренние итоги

- В общий чат каждой лиги приходит один итог игрового дня: результаты, игрок дня, лидер и короткий AI-комментарий.
- Каждый участник получает отдельное личное утреннее сообщение со своими очками, попаданиями и текущим местом в выбранной лиге.
- По умолчанию тон — `Без пощады`: футбольная ирония о прогнозах без мата, оскорблений и переходов на личность.

### Настройки тона

- В профиле участник может выбрать личный стиль сообщений: `Без пощады`, `Ирония`, `Спокойно` или `Цифры`.
- Владелец/администратор лиги может выбрать тон утренних итогов общего Telegram-чата в разделе `Лиги → Управлять лигой`.

## Установка

1. Сделайте резервную копию Railway PostgreSQL.
2. В Railway → PostgreSQL → Query выполните SQL из файла:

   ```text
   db/migrations/020_add_gamification_and_ai_messages.sql
   ```

3. Проверьте, что в Railway задан `OPENAI_API_KEY`.
4. Распакуйте проект, выполните commit/push и дождитесь Railway-деплоя.
5. Полностью закройте и снова откройте PWA, чтобы загрузилась версия 2.8.55.

## Новые / рекомендуемые переменные окружения

```env
# Достаточно существующего ключа. Без ключа сообщения используют шаблонный текст.
OPENAI_API_KEY=...

# Необязательно: модель для коротких сообщений Отца.
OPENAI_GAMIFICATION_MODEL=gpt-5.5-mini

# Необязательно: выключить AI-комментарии, оставив все игровые функции и шаблонные сообщения.
OPENAI_GAMIFICATION_ENABLED=true
```

Для утренних сообщений уже используется действующий планировщик:

```env
DAILY_FACTS_ENABLED=true
DAILY_FACT_TARGET=both
DAILY_FACT_HOUR=9
DAILY_FACT_MINUTE=0
DAILY_FACT_TIMEZONE=Europe/Moscow
```

## Проверка после деплоя

1. В профиле выберите лигу в шапке и убедитесь, что показываются титул, форма и новые карточки достижений.
2. В `Лиги → Управлять лигой` сохраните нужный стиль общего чата.
3. Сразу после следующего автоматически внесённого результата проверьте личное сообщение участника.
4. Для ручной проверки утреннего общего сообщения администратор может использовать существующую команду `/admin_send_daily_fact_group`.
