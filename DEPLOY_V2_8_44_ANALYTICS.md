# Установка v2.8.44 — продуктовая аналитика

## 1. Сохраните резервную копию PostgreSQL

Перед изменением production-БД сделайте backup PostgreSQL в Railway.

## 2. Примените миграцию **до** деплоя приложения

В Railway откройте PostgreSQL → **Query** и выполните содержимое файла:

```text
db/migrations/019_add_product_analytics.sql
```

Миграция создаст таблицу `analytics_events` и индексы. Она безопасна для повторного запуска: в ней используются `IF NOT EXISTS`.

## 3. Замените файлы проекта и задеплойте

1. Распакуйте архив поверх проекта.
2. Выполните commit.
3. Отправьте изменения в GitLab / ветку, из которой Railway выполняет deploy.
4. Дождитесь успешного завершения build и запуска приложения.

## 4. Проверка после деплоя

При наличии доступа к консоли Railway выполните:

```bash
python scripts/check_db_schema.py
```

Ожидаемый результат:

```text
DB schema check passed.
```

Затем:

1. Откройте Mini App через Telegram или установленную PWA.
2. Несколько раз перейдите между разделами, откройте матч и сохраните тестовый прогноз.
3. Под администратором откройте **Администрирование → Аналитика**.
4. Через несколько секунд должны появиться первые события.

## Важно

Статистика начинает накапливаться только после установки v2.8.44; исторические действия за предыдущие версии не восстанавливаются.
